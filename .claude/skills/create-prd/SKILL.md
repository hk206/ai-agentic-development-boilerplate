# PRD作成

`$ARGUMENTS` に基づいてPRD（プロダクト要件定義書）を作成する。

## 手順

1. 指定された機能/プロダクトについてユーザーにヒアリングする（不明点があれば質問する）
2. 必要に応じて競合・市場調査を行う
3. PRDテンプレートに従ってドキュメントを作成する
4. `docs/prd/` ディレクトリに保存する（ファイル名: `[機能名].md`）

## PRDテンプレート

以下のテンプレートに従って作成すること: [templates/prd-template.md](templates/prd-template.md)

テンプレートの構造概要:

1. **背景・課題** — 現状の課題は何か、なぜ今この機能が必要か
2. **目的・ゴール** — この機能で達成したいこと、成功の定義
3. **ターゲットユーザー** — 主要ペルソナ、ユーザーの課題・ニーズ
4. **機能要件** — Must Have / Should Have / Could Have / Won't Have
5. **非機能要件** — パフォーマンス、セキュリティ、アクセシビリティ、スケーラビリティ
6. **成功指標（KPI）** — 定量的な目標値
7. **スケジュール** — マイルストーン
8. **リスクと対策** — 技術的リスク、ビジネスリスク、対策案
9. **前提・仮定** — `[要確認]` マークで不確実な事項を明示する
10. **参考資料** — 関連ドキュメント・URL

## 注意事項

- 不明点や仮定は明示的に `[要確認]` マークをつける
- ユーザー価値を常に中心に据える
- 技術的実現可能性にも触れること
- MVPスコープを意識し、過剰な機能追加を避ける
